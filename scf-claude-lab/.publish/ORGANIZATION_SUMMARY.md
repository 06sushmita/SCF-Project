## 🎓 Understanding the Architecture

### Level 1: Extraction
Reads PDFs, extracts clauses, cleans, deduplicates → Outputs `control.yaml`

### Level 2: Registry
Reads `control.yaml` → Creates registry with versioning & lifecycle → Outputs `registry_level2.json`

### Level 3: Policy-as-Code
Reads `control.yaml` → Auto-generates Rego policies & tests → Outputs policy packages

### Integration
Registry (L2) provides searchable control metadata  
Policies (L3) provide executable enforcement logic  
Both use same source: `control.yaml`
